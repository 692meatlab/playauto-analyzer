# 플레이오토 판매 분석기

> 플레이오토 주문/매출 데이터를 분석하는 Streamlit 대시보드

## 프로젝트 개요

| 항목 | 내용 |
|------|------|
| **프레임워크** | Streamlit |
| **데이터 처리** | Pandas |
| **시각화** | Plotly Express, Plotly Graph Objects |
| **저장소** | 로컬 pickle / GitHub API |
| **대상 사용자** | 육구이, 우주인 사업장 운영자 |

## 핵심 파일

```
app.py                    # 메인 Streamlit 앱 (대시보드 UI)
analyzers/
├── order_analyzer.py     # 주문/매출 분석 핵심 로직
└── sales_analyzer.py     # 매출 분석 보조
data/                     # 데이터 저장
reports/                  # 리포트 출력
```

## 실행 방법

```bash
streamlit run app.py
```

## 데이터 흐름

1. 엑셀 파일 업로드 (플레이오토 주문내역)
2. `OrderAnalyzer.load_excel()` → 전처리 및 사업장 분류
3. 분석 결과 → Streamlit 대시보드 표시
4. GitHub 또는 로컬에 데이터 저장

## 사업장 분류 규칙

| 사업장 | 계정 키워드 |
|--------|------------|
| 육구이 | `692meatlab`, `692` |
| 우주인 | `kms`, `sosin` |

## 하면 안 되는 것

### 타임존
- `datetime.now()` 사용 금지 → `now_kst()` 유틸 함수 사용
- ✅ 수정 완료 (2024-03-19)

### 시각화
- 막대 그래프 Y축은 반드시 0부터 시작 (`rangemode='tozero'`)
- 파이 차트는 5개 이하 항목만 (그 이상은 '기타'로 묶기)
- 3D 차트 사용 금지

### 데이터 처리
- 전처리 없이 분석 금지 (결측치, 이상치, 중복 체크 필수)
- `SELECT *` 패턴 금지 (필요 컬럼만)

### 보안
- GitHub 토큰 하드코딩 금지 → `st.secrets` 사용
- `.env` 파일 커밋 금지

## 성능 최적화

### 적용된 캐싱 (2024-03-19)
- `@st.cache_resource`: OrderAnalyzer 인스턴스 캐싱
- 효과: 4.7MB pickle 로드를 최초 1회만 수행

### 캐시 관련 함수
```python
_create_analyzer()      # 캐시된 analyzer 생성
get_analyzer()          # 캐시된 analyzer 반환
clear_analyzer_cache()  # 데이터 삭제 시 캐시 클리어
```

### 주의사항
- 데이터 삭제 시 `clear_analyzer_cache()` 호출 필수

## AI Native 발전 방향

### 현재 상태: AI-Assisted
- AI가 인사이트 생성 → 사용자가 보고 판단 → 사용자가 액션

### 목표: AI-Native
```
❌ 현재: 분석 결과 표시 → 사용자가 보고 판단 → 사용자가 프로모션 설정
✅ 목표: 분석 → 이상 감지 시 자동 알림 → 프로모션 자동 제안/적용 → 결과 보고
```

### 가능한 자동화
| 기능 | 현재 | AI-Native |
|------|------|-----------|
| 일일 리포트 | 수동 확인 | 매일 아침 슬랙/이메일 자동 발송 |
| 취소율 이상 | 대시보드 확인 | 임계치 초과 시 즉시 알림 |
| 재고 경고 | 없음 | 베스트셀러 재고 부족 예측 알림 |
| 프로모션 | 추천 표시 | 자동 등록 + 결과 보고 |
