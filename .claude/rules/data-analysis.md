# 데이터 분석 규칙

> 데이터 분석/처리가 있는 프로젝트에만 적용. 크롤링, ERP, 분석 프로젝트 등.

## 전처리 필수
데이터 분석 전 반드시:
- 결측치 처리 (dropna/fillna 명시적 선택)
- 이상치 탐지 (IQR 또는 z-score)
- 데이터 타입 불일치 검증
- 중복 행 체크

전처리 없이 분석 코드 작성 금지.

## 통계적 검증
단순 경향성만 제시 금지. 결론에는 반드시:
- 통계적 유의성 검정 (p-value, 신뢰구간)
- 또는 ML 모델 성능 지표 (precision, recall, F1)
- 샘플 크기와 한계 명시

## 재현성
- `random.seed()` / `np.random.seed()` 필수 설정
- 라이브러리 버전 명시 (requirements.txt 또는 주석)
- 데이터 소스 경로/날짜 기록

## 대용량 데이터
- DataFrame 전체 로딩 전 `nrows=100`으로 샘플 확인
- 8만건+ 처리 시 청크(chunk) 또는 배치 처리
- `SELECT *` 금지, 필요 컬럼만 조회 (LIMIT 필수)
