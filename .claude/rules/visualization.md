# 시각화 규칙

> 대시보드/차트가 있는 프로젝트에만 적용.

## 데이터 왜곡 금지
- Y축은 반드시 0부터 시작 (막대 그래프)
- 이중 Y축 사용 시 스케일 차이 명시
- 3D 차트 금지 (시각적 왜곡 유발)
- 파이 차트는 5개 이하 항목에서만

## 데이터-잉크 비율
Tufte 원칙: 불필요한 장식(Chartjunk) 제거.
- 배경 격자선은 연한 색 또는 제거
- 범례는 차트 안에 직접 라벨링 우선
- 색상은 최소한으로, 색맹 친화 팔레트 사용

## 대시보드 성능
- 8만건+ 데이터는 서버에서 집계 후 전송
- 클라이언트 전체 로딩 금지 → 페이지네이션 또는 가상 스크롤
- 차트 렌더링은 지연 로딩(lazy loading) 적용
