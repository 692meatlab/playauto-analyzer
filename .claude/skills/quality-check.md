# /quality-check - 품질 점수 평가

## 트리거
- "품질 체크", "quality check", "점수 확인", "score"

## 역할 구분
| 스킬 | 역할 | 출력 |
|------|------|------|
| `/verify` | 검증 명령어 실행 + 통과/실패 판정 | PASSED / FAILED |
| `/quality-check` | verify 결과 기반 점수 계산 + 개선 제안 | 0~100점 + 개선 목록 |

---

## 실행 순서

1. **프로젝트 유형 감지** - `pyproject.toml` → Python / `package.json` → Node.js
2. **verify 실행** - `/verify --detail` 실행, 각 항목(린트/타입/테스트/보안)의 출력 결과 수집
3. **코드 품질 채점** (65점) - verify 출력에서 에러 수, 통과율, 커버리지 % 추출 후 아래 테이블로 채점
4. **프로젝트 채점** (35점) - 파일 존재 여부와 내용 검토로 기능/문서/구조 채점
5. **결과 보고** - 보고 형식에 맞춰 총점과 항목별 점수 출력
6. **개선 제안** - 만점 미달 항목에 대해 구체적 액션과 예상 점수 상승 제시

## 프로젝트 품질 점수 (35점)

| 항목 | 만점 | 판단 방법 |
|------|------|-----------|
| 기능 완성도 | 20 | `PROJECT_SPEC.md`의 P0 체크리스트 완료율 확인. P0 100%+P1 80%+ → 20 / P0 완료 → 15 / P0 미완 → 5 / 스펙 없음 → 10 (코드 규모로 추정) |
| 문서화 | 10 | `CLAUDE.md` 존재+내용충실 → 5 / `README.md` 존재+내용충실 → 5. 파일 존재하지만 미흡하면 각 2점 |
| 프로젝트 구조 | 5 | src/tests 분리 → 2 / 설정 파일 존재 → 1 / .gitignore → 1 / scripts/ → 1 |

> **총점 = 코드 품질(65) + 프로젝트 품질(35) = 100점**

## 점수별 상태

| 점수 | 상태 | 판정 |
|------|------|------|
| 90+ | 훌륭함 | 프로덕션 배포 가능 |
| 80-89 | 양호 | 배포 가능, 개선 권장 |
| 70-79 | 주의 | 자동 수정 후 재평가 권장 |
| < 70 | 미흡 | 배포 전 필수 수정 |

## 옵션

| 옵션 | 설명 |
|------|------|
| `--detail` | verify 원본 출력 + 채점 근거 포함 |
| `--fix` | 자동 수정 가능 항목 실행 후 재채점 (린트: `ruff --fix` / `eslint --fix`, 포맷: `ruff format`) |
| `--json` | JSON 형식 출력 |

---

## 주의사항
- `--fix` 자동 수정은 최대 3회, 보안/로직/테스트는 대상 외
- 수정 전 `git stash` 권장
- 점수 자동화 스크립트: `scripts/quality_scorer.py` 참조
